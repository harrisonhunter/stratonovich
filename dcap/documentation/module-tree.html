Module Hierarchy

  • dcap: A Distributed Computation Architecture in Python
    • dcap.RunClient: Starts a dcap client.
    • dcap.RunServer: Starts a dcap server.
    • dcap.client: package containing all client side code
      • dcap.client.Client: Main client module
      • dcap.client.ClientSideTaskHandler: This module contains methods for dealing with tasks received from the server.
    • dcap.common: Code used by both server and client
      • dcap.common.LoggingModule: Module Responsible for handling logging on both client and server
      • dcap.common.UtilityFunctions: Various functions used by both the client as well as the server
    • dcap.server: Server code
      • dcap.server.Server: Main server module
      • dcap.server.ServerSideResultsProcessor: This module contains the code for updating the tasks after a result has been received from a client.
      • dcap.server.ServerSideTaskHandler: This module is responsible for dealing with tasks on the server side.
