 1  '''A Distributed Computation Architecture in Python 
 2   
 3  How to use: 
 4      - Replace tasks.txt with your own tasks. Each task should be on a sperate line. 
 5      - Each task should take the form of: 
 6          1. taskname,path/to/taskscript.py,path/to/datadirectory 
 7          2. taskname,path/to/taskscript.py, 
 8      - datadirectory is optional. If it's not being used, simply replace leave it blank, but leave the trailing comma, see 2 above  
 9      - To run, execute RunServer.py on the server and RunClient.py on the client. 
10   
11   
12  The command line arguments for RunServer are: 
13      - E{-}p Specify on which port the server should listen. Default is 4444 
14      - E{-}t Specify the file containing the tasks. Default is ./server/tasks/tasks.txt 
15      - E{-}c Completed tasks, specify the file containing subset of the tasks that should not be processed. Optional 
16      - E{-}n Specify a name that will be appended used to identify the results. Optional 
17   
18  The command line arguments for RunClient are: 
19      - E{-}ip IP4 address of server to connect to. Default is 127.0.0.1 
20      - E{-}p port that the server is listening on. Default is 4444 
21   
22   
23  Author: Alexander Waldin 
24  '''
